How to Call Products-Cron Admin Endpoints

A plain English guide for developers to access internal Kubernetes services securely.

Quick Summary

The products-cron service is private. To access it, you must use VPN + Port Forwarding to create a secure tunnel from your laptop to the cloud cluster.

Infrastructure Basics

The Kubernetes Cluster

A private group of computers (Nodes) running your applications (Pods). Nothing here is directly accessible from the internet.

How to Port Forward

The Connection Flow

sequenceDiagram participant Laptop as Your Laptop (8096) participant VPN as VPN Tunnel participant K8s as Kubernetes Cluster participant Pod as products-cron Pod (8080) Laptop->>VPN: curl http://localhost:8096/... VPN->>K8s: Authenticated Request K8s->>Pod: Forward to internal port 8080 Pod-->>Laptop: 200 OK (Response)

Step-by-Step Instructions

  1. 1

    Connect to VPN & Login

    Ensure you are on the corporate VPN and authenticated.

    az login
  2. 2

    Run the Script

    Navigate to the test utils folder and start the tunnel.

    cd Test-Utils/port-forward ./portforward-dev.sh
  3. 3

    Keep Terminal Open

    Wait for the "Forwarding port 8096" message. Do not close this window!

Admin Endpoints

Once port-forwarding is active, use these URLs in a new terminal window or Postman.

GET /test/fi-Rate/update-cached-markup
Port 8096

Reapplies config markup to all cached FI products. Useful when markup configuration has changed.

curl http://localhost:8096/test/fi-Rate/update-cached-markup
GET /test/fi-Rate/refresh
Port 8096

Triggers the cron job manually immediately (normally runs every 5 mins).

curl http://localhost:8096/test/fi-Rate/refresh
POST /test/fi-Rate/expire-products
Port 8096

Forces products for a specific state to expire (backdates them 90 days), forcing a fresh pull on next run.

curl -X POST http://localhost:8096/test/fi-Rate/expire-products \ -H "Content-Type: application/json" \ -d '{"dealerState": "FL"}'
