Flow Overview

01

Inventory Delta / Initial Cache

  1. Vehicle ADD/UPDATE message arrives (Azure Service Bus).
  2. VehicleInventoryTopicReceiver calls PenMediator.getVehicleServiceContractRate().
  3. PEN returns dealerCostInCents, retailPriceInCents, rateExist, etc.
  4. PenMediator applies the pricing rule (markup or override) and returns a PenFiProduct.
  5. Result is written to MongoDB fi-rate as vehicleServiceContractRate.
02

Scheduled Cron Refresh

  1. FIRateRefresh picks eligible cached documents (status ACTIVE, nextRefreshDate <= now).
  2. Non-FL states: FIRateService.updateCachedMarkup() reapplies markup to stored dealerCostInCents.
  3. FL states: Cron leaves VRC unchanged (it already holds PEN retail); refresh only updates T&W and VLP.
03

API Read Path

  1. /estimator/fi-products or /productsAdmin/* handlers fetch cache via FIService.
  2. FIService.getVehicleServiceContractPenProduct() mirrors the same rule when mapping to responses, ensuring both cached and on-demand pricing remain consistent.

Configuration

# library/src/main/resources/application.yml products: cost: vehicle-service-contract: markup-amount: 150000 # $1,500 markup over PEN dealer cost retail-price-override-for-states: - "FL" # States that should use retail price only

ProductsMarkupConfig.VehicleServiceContract (library/.../services/Config.kt) binds these values.

Admin & Troubleshooting

Admin Endpoints

Update Cached Markup
curl http://localhost:8096/test/fi-Rate/update-cached-markup

Requires port-forwarding or cluster access.

Florida caveat: The admin endpoint does not recompute VRC for FL; it already stores PEN retail. Changing markup-amount has no impact on FL until inventory events or manual adjustments occur.

Inspect Cache

Summary

  • VRC pricing is PEN-driven, with Driveway markup layered on top.
  • Florida (or other configured states) bypass markup and use PEN retail to satisfy local regulations.
  • Configuration lives in application.yml; business logic in PenMediator and FIService keeps cache, cron, and API responses aligned.
  • Admin tooling (update-cached-markup, port-forward script) lets you reapply markups when config changes.
