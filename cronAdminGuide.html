Why is Cron Hidden?

Security Risk

The cron service contains dangerous endpoints like /test/expire-products and has no internal authentication.

If exposed on the Gateway, anyone on the internet could wipe our pricing cache.

Service Type On Gateway? Example
api YES products-api
cron NO products-cron
headless NO inventory-receiver

Prerequisites

Ensure you have the tools installed.

brew install azure-cli kubectl brew install Azure/kubelogin/kubelogin az login
2

Run the Script

Connect to VPN, then run the helper script.

cd Test-Utils/port-forward ./portforward-dev.sh
Wait for "Forwarding port 8096". Do not close this terminal.

Admin Endpoints

Once the tunnel is open on port 8096, use these commands in a new terminal.

GET /test/fi-Rate/update-cached-markup

Reapplies configuration markup to all cached FI products. Use this after changing application.yml values.

curl http://localhost:8096/test/fi-Rate/update-cached-markup
GET /test/fi-Rate/refresh

Manually triggers the cron job batch immediately (bypassing the 5-minute schedule).

curl http://localhost:8096/test/fi-Rate/refresh
POST /test/fi-Rate/expire-products

Forces products for a specific state to expire (backdates 90 days), forcing a full re-fetch from PEN.

curl -X POST http://localhost:8096/test/fi-Rate/expire-products \ -H "Content-Type: application/json" \ -d '{"dealerState": "FL"}'

The Universal Chart Type

This setting in pipelines/dev.yml determines visibility.

pipelines/dev.yml - name: products-cron universalChartType: cron # ◄── HIDDEN - name: products-api universalChartType: api # ◄── PUBLIC

Cron Schedule

Defined in values-dev.yaml.

values-dev.yaml # 5 minutes in ms FI_RATE_REFRESH_CRON_INTERVAL: "300000" # Cron expression CRON_RATE_EXPRESSION: "0 0/5 * * * *"
